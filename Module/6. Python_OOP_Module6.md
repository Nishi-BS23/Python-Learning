## Introduction to OOP

### What is Object-Oriented Programming?

Object-Oriented Programming (OOP) is a programming paradigm based on the concept of "objects," which can contain data (attributes) and code (methods). Instead of writing programs as a sequence of instructions, OOP organizes code into reusable, self-contained objects.

Think of OOP like building with LEGO blocks:
- Each LEGO piece is an object with specific properties (color, shape, size)
- Pieces can be combined to create complex structures
- You can reuse pieces in different projects
- Each piece knows how to connect with others

### Key Concepts

- **Class**: A blueprint or template for creating objects. It defines the attributes and methods that all objects of that type will have.
- **Object**: An instance of a class. It's a specific realization of the class blueprint.
- **Attributes**: Variables that store data about the object (also called properties or fields).
- **Methods**: Functions that define the behavior of the object. They operate on the object's attributes.

**Real-world analogy**:
- **Class**: "Car" (the blueprint)
- **Object**: A specific car like "Alice's red Toyota"
- **Attributes**: color, model, year, speed
- **Methods**: drive(), brake(), accelerate(), get_fuel()

### Why OOP is Useful in Python

OOP offers several advantages:

- **Organization**: Groups related data and behavior together
- **Reusability**: Write code once, reuse it in multiple places
- **Modularity**: Break complex problems into smaller, manageable pieces
- **Encapsulation**: Hide internal details and expose only what's needed
- **Inheritance**: Create new classes based on existing ones
- **Polymorphism**: Use the same interface for different data types
- **Maintainability**: Easier to update and fix code

Python is a multi-paradigm language, meaning it supports OOP alongside procedural and functional programming styles.

## Classes and Objects

### Defining a Class Using class Keyword

A class is defined using the `class` keyword followed by the class name and a colon. By convention, class names use CamelCase (each word starts with a capital letter).

**Basic Syntax**:
```python

    pass
```

**Example**:

Creating an object (also called instantiation) is done by calling the class name like a function.


Each object is a unique instance with its own memory address.

## Attributes

Attributes are variables that belong to an object. They store data about the object's state.

### Instance Attributes vs Class Attributes

#### Instance Attributes
- Belong to a specific object (instance)
- Defined inside methods (usually `__init__`)
- Each object has its own copy
- Accessed using `self.attribute_name`

#### Class Attributes
- Belong to the class itself
- Shared by all instances
- Defined directly in the class body
- Accessed using `ClassName.attribute_name` or `self.attribute_name`

### How to Define and Access Attributes

```python
class Car:
    # Class attribute (shared by all cars)
    wheels = 4
    
    def __init__(self, color, model):
        # Instance attributes (unique to each car)
        self.color = color
        self.model = model

# Create objects
car1 = Car("red", "Toyota")
car2 = Car("blue", "Honda")

# Access instance attributes
print(car1.color)  # Output: red
print(car2.color)  # Output: blue

# Access class attribute
print(car1.wheels)  # Output: 4
print(car2.wheels)  # Output: 4
print(Car.wheels)   # Output: 4

# Modify class attribute (affects all instances)
Car.wheels = 6
print(car1.wheels)  # Output: 6
print(car2.wheels)  # Output: 6

# Modify instance attribute (only affects that instance)
car1.color = "green"
print(car1.color)  # Output: green
print(car2.color)  # Output: blue
```

## Methods

Methods are functions defined inside a class that operate on the object's attributes.

### Defining Methods Inside a Class

Methods are defined just like regular functions, but they're inside the class and take `self` as the first parameter.

```python
class Car:
    def __init__(self, color, model):
        self.color = color
        self.model = model
        self.speed = 0
    
    def accelerate(self):
        """Increase the car's speed."""
        self.speed += 10
        print(f"Speed increased to {self.speed} mph")
    
    def brake(self):
        """Decrease the car's speed."""
        if self.speed >= 5:
            self.speed -= 5
        else:
            self.speed = 0
        print(f"Speed decreased to {self.speed} mph")
    
    def get_speed(self):
        """Return the current speed."""
        return self.speed
```

### Using self Keyword

The `self` keyword refers to the current instance of the class. It's how methods access the object's attributes and other methods.

- `self` is always the first parameter in instance methods
- Python automatically passes the object as `self` when you call a method
- You don't pass `self` explicitly when calling methods

```python
class Car:
    def __init__(self, color):
        self.color = color
    
    def describe(self):
        # self refers to the object calling this method
        print(f"This car is {self.color}")
    
    def compare(self, other_car):
        # Compare this car (self) with another car
        if self.color == other_car.color:
            print("Both cars are the same color!")
        else:
            print("Different colors")

car1 = Car("red")
car2 = Car("blue")

car1.describe()  # self is automatically set to car1
car2.describe()  # self is automatically set to car2

car1.compare(car2)  # self=car1, other_car=car2
```

## Inheritance

### Concept of Parent and Child Classes

Inheritance allows a class (child class) to inherit attributes and methods from another class (parent class). This promotes code reuse and creates a hierarchical relationship between classes.

**Key Concepts**:
- **Parent Class (Base Class)**: The class being inherited from
- **Child Class (Derived Class)**: The class that inherits from the parent
- Child classes get all attributes and methods from the parent
- Child classes can add new attributes and methods
- Child classes can override (change) parent methods

**Real-world analogy**:
- Parent: "Vehicle" (has wheels, can move)
- Child: "Car" (inherits from Vehicle, adds specific car features)
- Child: "Bicycle" (inherits from Vehicle, adds specific bicycle features)

### How to Inherit Using class Child(Parent)

```python
class ParentClass:
    # Parent class attributes and methods
    pass

class ChildClass(ParentClass):
    # Child class inherits from ParentClass
    # Can add new attributes and methods
    # Can override parent methods
    pass
```

### Example: ElectricCar Inherits from Car

```python
class Car:
    """Parent Car class."""
    
    def __init__(self, color, model, year):
        self.color = color
        self.model = model
        self.year = year
        self.fuel_level = 100
    
    def drive(self):
        if self.fuel_level > 0:
            print(f"{self.color} {self.model} is driving!")
            self.fuel_level -= 10
        else:
            print("Out of fuel!")
    
    def refuel(self):
        self.fuel_level = 100
        print("Refueled to 100%")

class ElectricCar(Car):
    """Child ElectricCar class that inherits from Car."""
    
    def __init__(self, color, model, year, battery_capacity):
        # Call parent's __init__ to initialize inherited attributes
        super().__init__(color, model, year)
        # Add new attribute specific to ElectricCar
        self.battery_capacity = battery_capacity
        self.battery_level = 100
        self.fuel_level = 0  # Electric cars don't use fuel
    
    def charge(self):
        """New method specific to ElectricCar."""
        self.battery_level = 100
        print("Charged to 100%")
    
    def drive(self):
        """Override parent's drive method."""
        if self.battery_level > 0:
            print(f"{self.color} {self.model} is driving silently!")
            self.battery_level -= 10
        else:
            print("Battery depleted! Please charge.")
    
    def get_range(self):
        """New method to calculate driving range."""
        return self.battery_capacity * (self.battery_level / 100)

# Create and use objects
regular_car = Car("red", "Toyota", 2020)
electric_car = ElectricCar("blue", "Tesla", 2021, 300)

print("Regular Car:")
regular_car.drive()
regular_car.drive()
regular_car.refuel()

print("\nElectric Car:")
electric_car.drive()
electric_car.drive()
electric_car.charge()
print(f"Range: {electric_car.get_range()} miles")
```

**Output**:
```
Regular Car:
red Toyota is driving!
red Toyota is driving!
Refueled to 100%

Electric Car:
blue Tesla is driving silently!
blue Tesla is driving silently!
Charged to 100%
Range: 300.0 miles
```

### Overriding Methods in Child Classes

When a child class defines a method with the same name as a parent class method, the child's version overrides the parent's version.

```python
class Animal:
    def speak(self):
        print("Some generic animal sound")

class Dog(Animal):
    def speak(self):
        print("Woof! Woof!")

class Cat(Animal):
    def speak(self):
        print("Meow!")

class Cow(Animal):
    # Does not override, uses parent's method
    pass

# Create objects
animals = [Animal(), Dog(), Cat(), Cow()]

for animal in animals:
    animal.speak()
```

**Output**:
```
Some generic animal sound
Woof! Woof!
Meow!
Some generic animal sound
```

## Encapsulation and Polymorphism (Basic Introduction)

### Encapsulation: Private Attributes Using _ or __

Encapsulation is the practice of hiding internal details of how an object works and only exposing what's necessary. In Python, we use naming conventions to indicate that attributes should be treated as private.

#### Single Underscore (_)
- Indicates "protected" attribute (convention only)
- Still accessible, but signals "don't touch this directly"

#### Double Underscore (__)
- Indicates "private" attribute (name mangling)
- Harder to access from outside the class

```python
class BankAccount:
    def __init__(self, account_number, balance):
        self.account_number = account_number  # Public
        self._balance = balance              # Protected (convention)
        self.__pin = "1234"                  # Private (name mangling)
    
    def deposit(self, amount):
        if amount > 0:
            self._balance += amount
            print(f"Deposited ${amount}. New balance: ${self._balance}")
    
    def get_balance(self):
        """Public method to access protected balance."""
        return self._balance
    
    def verify_pin(self, pin):
        """Public method to verify private pin."""
        return self.__pin == pin

account = BankAccount("12345", 1000)

# Public attribute - accessible
print(account.account_number)  # Output: 12345

# Protected attribute - accessible but discouraged
print(account._balance)  # Output: 1000

# Private attribute - name mangling makes it harder
# print(account.__pin)  # This would raise AttributeError
print(account._BankAccount__pin)  # Output: 1234 (access via mangled name)

# Use public methods instead
account.deposit(500)
print(f"Balance: ${account.get_balance()}")
print(f"PIN correct: {account.verify_pin('1234')}")
```

### Polymorphism: Same Method Name in Different Classes

Polymorphism allows different classes to have methods with the same name but different implementations. This enables you to write code that works with objects of different classes uniformly.

```python
class Shape:
    def area(self):
        pass
    
    def perimeter(self):
        pass

class Rectangle(Shape):
    def __init__(self, width, height):
        self.width = width
        self.height = height
    
    def area(self):
        return self.width * self.height
    
    def perimeter(self):
        return 2 * (self.width + self.height)

class Circle(Shape):
    def __init__(self, radius):
        self.radius = radius
    
    def area(self):
        return 3.14159 * self.radius ** 2
    
    def perimeter(self):
        return 2 * 3.14159 * self.radius

# Polymorphism: same method calls, different implementations
shapes = [
    Rectangle(5, 3),
    Circle(4),
    Rectangle(2, 2)
]

for shape in shapes:
    print(f"Shape: {shape.__class__.__name__}")
    print(f"  Area: {shape.area():.2f}")
    print(f"  Perimeter: {shape.perimeter():.2f}")
    print()
```

**Output**:
```
Shape: Rectangle
  Area: 15.00
  Perimeter: 16.00

Shape: Circle
  Area: 50.27
  Perimeter: 25.13

Shape: Rectangle
  Area: 4.00
  Perimeter: 8.00
```

## Special Methods

### __str__ and __repr__ for Printing Objects

Special methods (also called magic methods or dunder methods) allow you to define how objects behave with built-in operations.

#### __str__ Method
- Returns a user-friendly string representation
- Called by `str()` and `print()`
- Should be readable and informative

#### __repr__ Method
- Returns a developer-friendly string representation
- Called by `repr()` and in interactive sessions
- Should ideally be a valid Python expression that recreates the object

### Example: Customize Car Object Output

```python
class Car:
    def __init__(self, color, model, year):
        self.color = color
        self.model = model
        self.year = year
    
    def __str__(self):
        """User-friendly string representation."""
        return f"{self.year} {self.color} {self.model}"
    
    def __repr__(self):
        """Developer-friendly string representation."""
        return f"Car('{self.color}', '{self.model}', {self.year})"

car = Car("red", "Toyota", 2020)

# Using print() calls __str__
print(car)  # Output: 2020 red Toyota

# Using str() calls __str__
print(str(car))  # Output: 2020 red Toyota

# Using repr() calls __repr__
print(repr(car))  # Output: Car('red', 'Toyota', 2020)

# In interactive Python, just typing the variable shows __repr__
# car would display: Car('red', 'Toyota', 2020)
```

**Other Useful Special Methods**:
```python
class Number:
    def __init__(self, value):
        self.value = value
    
    def __add__(self, other):
        """Define behavior for + operator."""
        return Number(self.value + other.value)
    
    def __eq__(self, other):
        """Define behavior for == operator."""
        return self.value == other.value
    
    def __len__(self):
        """Define behavior for len()."""
        return len(str(self.value))

n1 = Number(5)
n2 = Number(10)

n3 = n1 + n2  # Uses __add__
print(n3.value)  # Output: 15

print(n1 == n2)  # Uses __eq__, Output: False
```

```python
class BankAccount:
    """A simple bank account class."""
    
    def __init__(self, account_number, owner, balance=0):
        self.account_number = account_number
        self.owner = owner
        self._balance = balance  # Protected attribute
    
    def deposit(self, amount):
        """Deposit money into the account."""
        if amount > 0:
            self._balance += amount
            print(f"Deposited ${amount}. New balance: ${self._balance}")
        else:
            print("Deposit amount must be positive.")
    
    def withdraw(self, amount):
        """Withdraw money from the account."""
        if amount > self._balance:
            print("Insufficient funds!")
        elif amount <= 0:
            print("Withdrawal amount must be positive.")
        else:
            self._balance -= amount
            print(f"Withdrew ${amount}. New balance: ${self._balance}")
    
    def check_balance(self):
        """Check the current balance."""
        print(f"Account: {self.account_number}")
        print(f"Owner: {self.owner}")
        print(f"Balance: ${self._balance}")
        return self._balance

# Create and use a bank account

account.deposit(500)
account.withdraw(200)
account.withdraw(2000)  # Insufficient funds
account.check_balance()
```

```python
class SavingsAccount(BankAccount):
    """Savings account that earns interest."""
    
    def __init__(self, account_number, owner, balance=0, interest_rate=0.02):
        # Call parent's __init__
        super().__init__(account_number, owner, balance)

        """Apply interest to the balance."""

        """Custom string representation."""
```

```python
class Person:
```

```python
    def __init__(self, name, age):
        self.name = name
        self.age = age
    
    def greet(self):
        print(f"Hello, I'm {self.name}.")

class Employee(Person):
    """Employee class that inherits from Person."""

        # Call parent's __init__
        super().__init__(name, age)
        # Add employee-specific attributes
        self.salary = salary
        self.position = position

        """Display employee salary information."""

        print(f"Position: {self.position}")

        """Give the employee a raise."""
        if amount > 0:
            self.salary += amount
            print(f"{self.name} received a raise of ${amount:,}!")
            print(f"New salary: ${self.salary:,}")
        else:
            print("Raise amount must be positive.")
    
    def greet(self):
        """Override parent's greet method."""
        print(f"Hello, I'm {self.name}, a {self.position} at this company.")

# Create and use employee objects
```

### Exercise 4: Override __str__ Method to Print Object Details

Create a class with a custom `__str__` method to provide meaningful output when printing the object.

```python
# Solution
class Book:
    """A book class with title, author, and year."""
    
    def __init__(self, title, author, year, isbn):
        self.title = title
        self.author = author
        self.year = year
        self.isbn = isbn
    
    def __str__(self):
        """Return a formatted string representation."""
        return f"'{self.title}' by {self.author} ({self.year}) [ISBN: {self.isbn}]"
    
    def __repr__(self):
        """Return a developer-friendly representation."""
        return f"Book(title='{self.title}', author='{self.author}', year={self.year}, isbn='{self.isbn}')"

# Create and use book objects
book1 = Book("Python Crash Course", "Eric Matthes", 2019, "978-1593279288")
book2 = Book("Clean Code", "Robert C. Martin", 2008, "978-0132350884")

# Using print() calls __str__

print(book2)

print()
print("repr(book1):", repr(book1))

print()
books = [book1, book2]
for book in books:
    print(book)  # Each book uses __str__
```

### Common Mistakes to Avoid

- **Forgetting `self`**: Always include `self` as the first parameter in instance methods
- **Confusing class and instance attributes**: Remember class attributes are shared, instance attributes are unique
- **Not calling `super().__init__()`**: When inheriting, remember to initialize parent attributes
- **Overusing inheritance**: Use composition over inheritance when appropriate
- **Making everything public**: Use encapsulation to hide implementation details
- **Ignoring naming conventions**: Use CamelCase for classes, snake_case for methods/attributes
- **Not using `__str__`**: Always provide a readable string representation for debugging
