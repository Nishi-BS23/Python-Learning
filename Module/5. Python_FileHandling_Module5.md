## Introduction to File Handling

### What is File Handling and Why is it Important?

File handling is the process of reading from and writing to files stored on your computer's disk. It allows your Python programs to:

- **Persist Data**: Save data so it remains available after the program ends
- **Read Configuration**: Load settings and preferences from files
- **Process Data**: Analyze data stored in files (logs, reports, datasets)
- **Generate Reports**: Create output files with program results
- **Share Data**: Exchange information between different programs

Without file handling, all data would be lost when a program stops running. File handling bridges the gap between your program's memory and permanent storage.

### Types of Files

Python can work with various file types, but the most common for beginners are:

#### Text Files
- Plain text files with `.txt` extension
- Contain human-readable characters
- Simple structure - just lines of text
- Easy to create and edit with any text editor

#### CSV Files
- Comma-Separated Values files with `.csv` extension
- Store tabular data (like spreadsheets)
- Each line represents a row, commas separate columns
- Widely used for data exchange between applications

**Other File Types** (for future learning):
- JSON files - for structured data
- XML files - for markup data
- Binary files - images, audio, video, executables

### Modes of File Handling

File modes determine what operations you can perform on a file:

- **Read Mode (`'r'`)**: Only read from the file (file must exist)
- **Write Mode (`'w'`)**: Write to the file (creates new or overwrites existing)
- **Append Mode (`'a'`)**: Add to the end of the file (preserves existing content)
- **Read-Write Mode (`'r+'`)**: Both read and write (file must exist)

Choosing the right mode is crucial to avoid accidental data loss!

## Opening and Closing Files

### open(filename, mode) Syntax

The `open()` function is used to open a file. It returns a file object that provides methods for reading and writing.

**Basic Syntax**:
```python
file_object = open(filename, mode)
# Perform file operations
file_object.close()
```

### File Modes

| Mode | Description | File Must Exist? | Action if File Exists |
|------|-------------|------------------|----------------------|
| `'r'` | Read only | Yes | Opens for reading |
| `'w'` | Write only | No | Creates new or overwrites |
| `'a'` | Append only | No | Creates new or appends |
| `'r+'` | Read and write | Yes | Opens for both |
| `'w+'` | Read and write | No | Creates or overwrites |
| `'a+'` | Read and append | No | Creates or appends |

### Closing Files Using close()

It's important to close files after you're done to:
- Free up system resources
- Ensure all data is written to disk
- Allow other programs to access the file

```python
# Opening and closing a file
file = open("example.txt", "r")
content = file.read()
print(content)
file.close()
```

**Warning**: If you forget to close a file, it may cause issues:
- Data might not be saved
- The file remains locked
- System resources are wasted

### Using with Statement for Automatic Closing

The `with` statement is the recommended way to handle files. It automatically closes the file, even if an error occurs.

**Syntax**:
```python
with open(filename, mode) as file_object:
    # File operations here
# File is automatically closed after this block
```

**Example**:
```python
# Using with statement (recommended)
with open("example.txt", "r") as file:
    content = file.read()
    print(content)
# File is automatically closed here
```

**Advantages of `with`**:
- Automatic file closing (no need to call `close()`)
- Exception-safe (closes even if errors occur)
- Cleaner, more readable code
- Prevents resource leaks

## Reading Files

### read() Method

The `read()` method reads the entire file content as a single string.

```python
# Create a sample file first
with open("sample.txt", "w") as file:
    file.write("Hello, World!\nThis is a sample file.\nIt has multiple lines.")

# Reading the entire file
with open("sample.txt", "r") as file:
    content = file.read()
    print("Entire file:")
    print(content)
```

**Output**:
```
Entire file:
Hello, World!
This is a sample file.
It has multiple lines.
```

**Reading with size limit**:
```python
with open("sample.txt", "r") as file:
    first_10_chars = file.read(10)
    print("First 10 characters:", first_10_chars)
```

### readline() Method

The `readline()` method reads one line at a time.

```python
with open("sample.txt", "r") as file:
    line1 = file.readline()
    line2 = file.readline()
    
    print("First line:", line1, end='')
    print("Second line:", line2, end='')
```

**Output**:
```
First line: Hello, World!
Second line: This is a sample file.
```

### readlines() Method

The `readlines()` method reads all lines and returns them as a list.

```python
with open("sample.txt", "r") as file:
    lines = file.readlines()
    print("Lines as list:", lines)
    
    # Process each line
    print("\nProcessing each line:")
    for line in lines:
        print(f"Line: {line.strip()}")
```

**Output**:
```
Lines as list: ['Hello, World!\n', 'This is a sample file.\n', 'It has multiple lines.']

Processing each line:
Line: Hello, World!
Line: This is a sample file.
Line: It has multiple lines.
```

### Iterating Over a File Line by Line

The most efficient way to read large files is to iterate directly over the file object.

```python
with open("sample.txt", "r") as file:
    for line in file:
        print(f"Line: {line.strip()}")
```

**Output**:
```
Line: Hello, World!
Line: This is a sample file.
Line: It has multiple lines.
```

**Advantages**:
- Memory efficient (doesn't load entire file)
- Clean and readable
- Works with files of any size

## Writing Files

### write() Method

The `write()` method writes a string to the file. If the file doesn't exist in write mode, it creates a new one.

```python
# Writing to a file (creates or overwrites)
with open("output.txt", "w") as file:
    file.write("Hello, World!\n")
    file.write("This is a new file.\n")
    file.write("Writing is fun!")

print("File created successfully!")
```

### writelines() Method

The `writelines()` method writes a list of strings to the file. It doesn't add newlines automatically.

```python
lines = ["First line\n", "Second line\n", "Third line\n"]

with open("lines.txt", "w") as file:
    file.writelines(lines)

print("Lines written to file!")
```

**Note**: You need to include `\n` in each string if you want separate lines.

### Overwriting vs Appending

#### Overwriting (Write Mode `'w'`)
```python
# First write
with open("demo.txt", "w") as file:
    file.write("Original content")

print("After first write:")
with open("demo.txt", "r") as file:
    print(file.read())

# Overwrite with new content
with open("demo.txt", "w") as file:
    file.write("New content - old content is gone!")

print("\nAfter overwrite:")
with open("demo.txt", "r") as file:
    print(file.read())
```

**Output**:
```
After first write:
Original content

After overwrite:
New content - old content is gone!
```

#### Appending (Append Mode `'a'`)
```python
# First write
with open("demo.txt", "w") as file:
    file.write("First line\n")

print("After first write:")
with open("demo.txt", "r") as file:
    print(file.read())

# Append more content
with open("demo.txt", "a") as file:
    file.write("Second line\n")
    file.write("Third line\n")

print("\nAfter appending:")
with open("demo.txt", "r") as file:
    print(file.read())
```

**Output**:
```
After first write:
First line

After appending:
First line
Second line
Third line
```

## Practical Examples

### Example 1: Count the Number of Words in a Text File

```python
# Create a sample file
with open("wordcount.txt", "w") as file:
    file.write("""Python is an amazing programming language.
It is easy to learn and powerful.
Many people love Python for its simplicity.
Python can be used for web development, data science, and automation.""")

# Function to count words
def count_words(filename):
    """Count the total number of words in a file."""
    word_count = 0
    
    with open(filename, "r") as file:
        for line in file:
            # Split line into words
            words = line.split()
            word_count += len(words)
    
    return word_count

# Count words
total_words = count_words("wordcount.txt")
print(f"Total words in file: {total_words}")

# Alternative: Count unique words
def count_unique_words(filename):
    """Count unique words in a file."""
    unique_words = set()
    
    with open(filename, "r") as file:
        for line in file:
            words = line.split()
            unique_words.update(words)
    
    return len(unique_words)

unique_count = count_unique_words("wordcount.txt")
print(f"Unique words in file: {unique_count}")
```

**Output**:
```
Total words in file: 28
Unique words in file: 22
```

## Handling Exceptions in File Operations

### Using try and except for Errors

File operations can fail for various reasons:
- File doesn't exist
- Permission denied
- Disk full
- File is in use by another program

Exception handling ensures your program doesn't crash and provides helpful error messages.

### Basic Exception Handling

```python
try:
    with open("nonexistent.txt", "r") as file:
        content = file.read()
except FileNotFoundError:
    print("Error: The file does not exist!")
```

### Example: Handle File Not Found Error

```python
def read_file_safely(filename):
    """Read a file with proper error handling."""
    try:
        with open(filename, "r") as file:
            content = file.read()
            print("File read successfully!")
            return content
    except FileNotFoundError:
        print(f"Error: File '{filename}' not found!")
        return None
    except PermissionError:
        print(f"Error: Permission denied for '{filename}'!")
        return None
    except Exception as e:
        print(f"An unexpected error occurred: {e}")
        return None

# Test with existing file
print("Test 1: Reading existing file:")
read_file_safely("sample.txt")

# Test with non-existent file
print("\nTest 2: Reading non-existent file:")
read_file_safely("nonexistent.txt")
```

**Output**:
```
Test 1: Reading existing file:
File read successfully!

Test 2: Reading non-existent file:
Error: File 'nonexistent.txt' not found!
```

### Comprehensive Exception Handling for Writing

```python
def write_file_safely(filename, content):
    """Write to a file with proper error handling."""
    try:
        with open(filename, "w") as file:
            file.write(content)
        print(f"Successfully wrote to '{filename}'")
        return True
    except PermissionError:
        print(f"Error: No permission to write to '{filename}'")
        return False
    except IsADirectoryError:
        print(f"Error: '{filename}' is a directory, not a file")
        return False
    except Exception as e:
        print(f"An unexpected error occurred: {e}")
        return False

# Test writing
write_file_safely("test_output.txt", "This is a test.")
write_file_safely("/root/protected.txt", "This won't work")  # Permission error
```

**Output**:
```
Successfully wrote to 'test_output.txt'
Error: No permission to write to '/root/protected.txt'
```

### Using else and finally with try-except

```python
try:
    with open("sample.txt", "r") as file:
        content = file.read()
except FileNotFoundError:
    print("File not found!")
else:
    # This runs if no exception occurred
    print(f"File has {len(content)} characters")
finally:
    # This always runs, regardless of exceptions
    print("File operation attempted")
```

## Working with CSV Files (Optional for Beginners)

### Introduction to CSV Files

CSV (Comma-Separated Values) is a simple file format for storing tabular data. Each line represents a row, and values are separated by commas.

**Example CSV file (students.csv)**:
```
name,age,grade
Alice,20,A
Bob,21,B
Charlie,19,A
Diana,22,B
```

CSV files are:
- Human-readable (can be opened in text editors)
- Compatible with spreadsheets (Excel, Google Sheets)
- Easy to import/export from databases
- Widely used for data exchange

### Using csv Module to Read and Write CSV Files

Python's built-in `csv` module makes working with CSV files easy.

#### Reading CSV Files

```python
import csv

# Create a sample CSV file
with open("students.csv", "w", newline="") as file:
    writer = csv.writer(file)
    writer.writerow(["name", "age", "grade"])
    writer.writerow(["Alice", 20, "A"])
    writer.writerow(["Bob", 21, "B"])
    writer.writerow(["Charlie", 19, "A"])
    writer.writerow(["Diana", 22, "B"])

# Reading CSV file
print("Reading CSV file:")
with open("students.csv", "r", newline="") as file:
    reader = csv.reader(file)
    
    for row in reader:
        print(row)
```

**Output**:
```
Reading CSV file:
['name', 'age', 'grade']
['Alice', '20', 'A']
['Bob', '21', 'B']
['Charlie', '19', 'A']
['Diana', '22', 'B']
```

#### Writing CSV Files

```python
import csv

# Data to write
students = [
    ["Eve", 23, "A"],
    ["Frank", 20, "C"],
    ["Grace", 21, "B"]
]

# Write to CSV file
with open("new_students.csv", "w", newline="") as file:
    writer = csv.writer(file)
    writer.writerow(["name", "age", "grade"])  # Header
    writer.writerows(students)  # Multiple rows

print("CSV file created successfully!")
```

def count_lines_with_word(filename, word):
    """Count lines containing a specific word."""
    count = 0
    matching_lines = []
    
    with open(filename, "r") as file:
        for line_num, line in enumerate(file, 1):

                matching_lines.append((line_num, line.strip()))

### Emphasize Using with Statement for Safety

The `with` statement is the recommended way to handle files in Python:

- **Automatic Closing**: Files close automatically when the block ends
- **Exception Safety**: Files close even if errors occur
- **Cleaner Code**: No need to remember `close()`
- **Resource Management**: Prevents resource leaks

**Always prefer**:
```python
with open("file.txt", "r") as file:
    content = file.read()
```

**Over**:
```python
file = open("file.txt", "r")
content = file.read()
file.close()  # Easy to forget!
```

### Common Mistakes to Avoid

- **Forgetting file mode**: Always specify the mode when opening files
- **Not using `with`**: Manual file closing is error-prone
- **Confusing read modes**: `'r'` requires file to exist, `'w'` overwrites
- **Ignoring encoding**: Specify encoding for non-ASCII text
- **Not handling exceptions**: File operations can fail
- **Forgetting newlines**: Remember `\n` when writing lines
- **Loading large files entirely**: Use iteration for big files
